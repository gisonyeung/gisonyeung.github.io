欢迎使用网页微信API

概述

本api提供简单模拟微信网页版包括用户登录、添加好友、聊天等功能，仅供广东工业大学研发中心前端考核使用。页面中部分素材源自网络，请不要用于商业用途。

当前版本：bata 0.2

接口的形式

接口一般由三部分组成，分别是域名、模块名称和接口名称

<项目地址>?action=<模块名>&method=<接口名>

域名

域名就是项目地址，你可以简单的看作是这个页面的地址。它可能是

http://cheewp.duapp.com/WeChat/

注意，由于项目地址可能会发生变动，所以建议大家用一个字符串来保存项目的地址。

模块名称

为了更好的区分各个接口，人为的将接口划分成了多个模块，如用户模块、好友模块、聊天模块等。模块名称就是他们的名称。原则上不区分大小写，但是请保证大小写一致。

模块名会包含在URL参数中，以action命名，如

<项目地址>?action=<模块名>

当然啦，如果不填的话，模块名默认是index，也就是你现在看到的页面，实际上是

<项目地址>?action=Index

需要用到的知识

在使用本接口之前，你需要掌握一些预备知识，他们包括http post、json、ajax等。 当然啦，为了应用好他们，你还需要学习javascript和jQuery。另外，如果能够了解session和cookie的话就更好啦，不排除哪天我会问你们哟。 这里会对上述知识做简单介绍，更多的内容需要大家自己　　上网找资料学习。

本次考核中，多数的接口使用方式都是采用前端post参数给后台，后台返回json对象的形式。

HTTP POST协议

在http协议里面最常使用两种传递参数的方式，他们分别是get和post。相比较而言，get比较方便，可以应对一些参数比较少、值比较短的情况。post比较安全，可以应对一些对安全性比较高的场合。在本次考核中，大部分的参数都会用post来提交。 不过有时候post和get其实是混合在一起使用的 :p

常见的提交post参数的方式主要有两种，一种是将表单(form)标签中的method属性设置成post，另一种则是通过jQuery或js进行提交。 在实际应用中，采用较早版本的jQuery可能会出现中文乱码的情况。

JSON对象

json对象是javascript世界中用来传递数据的一种字符串。这次考核中的接口大多都会返回一个json字符串，你需要自己想办法解析这个字符串，达到显示的目的。 因此，你需要学习如何解析json字符串。

Ajax加载

Ajax是异步加载页面的简称。 例如我们访问新浪微博就会发现，网页里面的内容并不是在一开始就被加载出来的。 而是等页面的架构全部就位之后才逐渐加载的。 这种分多次加载页面的展示方式就是异步加载。 为了实现这个功能，你可能需要学习如何用javascript实现ajax（w3school上有相关的介绍），如果实在受不了的话可以使用jQuery。

注意

如果发现没办法保持登录状态，请在post数据之前添加下面的代码

$.ajaxSetup({ xhrFields: { withCredentials: true }, crossDomain: true });

用户相关的接口

用户是一个聊天工具最核心的组成部分。这里将会提供包括注册、登录、修改用户信息等相关接口。

所有用户相关的接口都遵循以下面的url格式

<项目地址>?action=User&method=<接口名>

例如，登录的接口为

<项目地址>?action=User&method=login

注册

注册接口暂不开放

登录login

用于登录的接口。所有接口中，有相当一部分接口都是需要用户登录之后才可以正常使用的。当用户登录以后，服务器会记住用户的登录状态。

登陆后，如果长时间没有进行操作，服务器会认为用户已经下线。你可以通过下面的（checkIsLogin）接口检查自己现在还在不在线。同样的你也可以通过下线接口（logout）来主动下线。

接口地址

<项目地址>?action=User&method=login

请求参数

{
username:'admin',//字符串，用户名
password:'admin'//字符串，密码
}

返回参数

调用成功，则返回

{
"statue":"success",//字符串，表示成功状态，下同
"userid":"1" //字符串，服务器会返回用户的id，类似于QQ号
}

调用失败，则返回

{
"statue":"error",//字符串，表示失败状态，下同
"error":"password not set"//字符串，提示信息，如未设置密码
}

检查登录状态checkIsLogin

用于检查登录状态的接口.

接口地址

<项目地址>?action=User&method=checkIsLogin

请求参数

无

返回参数

如果目前处于登录状态，返回

{
"statue":"success"
}

如果已经处于登出状态，则返回

{
"statue":"error",
"error":"unlog"
}

下线，登出logout

用于下线的接口。

接口地址

<项目地址>?action=User&method=logout

请求参数

无

返回参数

只返回一句

{
"statue":"success"
}

修改信息update

用于修改自己的用户信息的接口。修改后可以调用获取用户信息getUserById借口进行检查。

接口地址

<项目地址>?action=User&method=update

请求参数

{
nickname,//字符串，用户的昵称
age,//整型，用户的年龄
sex,//单个字符，用户的性别用“男”“女”表示
introduction,//字符串，用户的自我介绍
email,//字符串，用户的email
}

返回参数

调用成功，则返回

{
"statue":"success"//字符串，表示成功状态，下同
}

调用失败，则返回

{
"statue":"error",//字符串，表示失败状态，下同
"error":"password not set"//字符串，提示信息，如未设置密码
}

获取用户信息getUserById

用于获取用户信息的接口。类似于使用QQ号码访问用户的信息。

接口地址

<项目地址>?action=User&method=getUserById

请求参数

{
userid:'1'//字符串，用户的帐号，类似与QQ号
}

返回参数

调用成功，则返回

{
"age": 50,//整型，用户的年龄
"email": "123@qq.com",//字符串，用户的email
"introduction": "我是王尼玛",//字符串，用户设置的自我介绍
"nickName": "群主",//字符串，用户的昵称
"sex": "",//单个字符，用户的性别
"userId": "1",//字符串，用户的id，类似于QQ号
}

调用失败，则返回

{
"statue":"error",//字符串，表示失败状态，下同
"error":"password not set"//字符串，提示信息，如未设置密码
}

好友相关的接口

好友关系是聊天系统中的重要关系。这里提供了获取好友列表、获取添加好友请求列表、添加查找好友等相关接口

所有好友相关的接口都遵循以下面的url格式

<项目地址>?action=Friend&method=<接口名>

例如，获取好友列表的接口为

<项目地址>?action=Friend&method=getFriendsList

获取好友列表getFriendsList

当用户登录了聊天工具之后，首先希望看到的就是好友列表啦。所以建议在登录完成之后调用一次这个接口。

需要注意的是，这个接口返回的数据是一个数组而不是前面说的对象。

另外由于这个接口返回的数据比较少（只有好友的id和备注名），所以你还要通过其他的接口（如getUserById）获取一下用户的其它信息。

接口地址

<项目地址>?action=Friend&method=getFriendsList

请求参数

无

返回参数

调用成功，则返回一个包含好友id和好友备注的json数组

[
{
"userid":"2db72f3e46",//字符串，用户的id，类似于QQ号
"remark":null//字符串，你给这个用户标记的备注，null代表没有备注
},
{
"userid":"asdjelncke",//字符串，用户的id，类似于QQ号
"remark":"师兄疯了"//字符串，你给这个用户标记的备注
}
]

调用失败，则返回

{
"statue":"error",//字符串，表示失败状态，下同
"error":"unlog"//字符串，提示信息，如未登录
}

可以修改好友的备注信息。

接口地址

<项目地址>?action=Friend&method=setRemark

请求参数

{

调用失败，则返回错误提示

{
"statue":"error",//字符串，表示失败状态，下同

接口地址

<项目地址>?action=Message&method=getUnreadUserId

无

返回参数
